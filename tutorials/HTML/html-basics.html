HTML basics

What is HTML?

HTML stands for Hypertext Markup Language and it was developed for formatting documents. It was originally designed to standardize formatting of scientific articles in the early days of the internet.

Naming your HTML file

When naming your html file there are a few things you should consider:

  • An html document name becomes part of the URL so when naming your file make sure your name is descriptive. This will also help you when it comes to maintenance.
  • Use only letters (a-z), numbers (0-9) and if needed underscores(_), hyphens(-) an periods(.).
  • While most operating systems allow you to use spaces in filenames you should not use spaces in the name of an html file.
  • You also should not start your html filenames with special characters or numbers. Some programing languages will not treat names starting with numbers as files and so will not load the page correctly.
  • Use lowercase letters. Web servers are case sensitive. It becomes easier to write your webpages if you don't have to remember whether you filename was upper or lower case.
  • Using hyphens to separate diferent words in your filename can help when search engines read your URL improving your search engine optimization (SEO).

What do I need in my HTML file?

You don't really need a lot for an HTML file to load.

The first line of your file needs the DOCTYPE tag <!DOCTYPE html>. This tells the browser that the document is HTML.

Next after DOCTYPE comes the opening html tag <html lang="en"> note that inside the tag we set the language attribute. Many element tags have attributes that we set when we use the opening tag. We will go over important attributes when learning about element tags. You are encouraged to research more at https://developer.mozilla.org/en-US/docs/Web/HTML/Element

we start the document header with the <head> tag.

The first element in the header is the meta tag that specifies the character encoding of the page <meta charset="utf-8">. This makes sure the document is displayed correctly.

After the meta tag come to the title element <title>. The text written between the opening and closing title tags will be displayed at the top of the browser when the page is loaded. In the example above it is simply the word title. This is also where we use our first closing tag. The closing tag looks almost exactly like the opening tag but has a / </title>.

That is all we need for the head section of the document. There are other things that we usually place in the head like links to stylesheets or scripts but we will discuss that when we talk about CSS and JavaScript. Since we aren't using them here we will use the closing tag </head> to indicate the head section is finished

Following the head we have the body of the document. This is the main section of the page and is where we will be putting our content. We open it with the <body> tag. The content placed between the opening and closing body tags is what we expect to see in the browser. In our example above we expect the browser to display the text "Hello World!". Normally we would place the text inside other element tags to allow styling and provide context to the content.

Finally we end the document with the closing HTML tag </html>

That's it! That's is all we need to create an HTML file that will load in the browser. It may not be an awesome website yet but it's a start! In the next lesson we will go into more HTML element tags and when to use them.
